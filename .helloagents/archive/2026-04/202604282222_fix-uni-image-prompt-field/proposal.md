## 1. 需求

### 背景
生成作品工作流执行原创配图时，Uni 图片生成接口返回 `HTTP 400`，错误体提示 `field Prompt is required`。Uni `/v1/images/generations` 的兼容协议实际使用小写 `prompt` 字段；该错误更符合“传入的提示词为空或缺失后被服务端按结构字段名 `Prompt` 报错”的表现。当前链路缺少本地强校验，空配图提示词会继续请求远端。

### 目标
修复原创配图提示词与图片生成入口的空值校验，确保生成作品工作流不会把空 `prompt` 发送到远端图片接口，并通过回归测试覆盖该行为。

### 约束条件
```yaml
时间约束: 无
性能约束: 不增加额外远程调用
兼容性约束: 不改变 ark/openai provider 行为
业务约束: 生成作品库写入链路保持不变
```

### 验收标准
- [ ] `uni` 文生图请求体继续使用小写 `prompt` 字段。
- [ ] 空图片提示词在本地被拦截，并返回可读错误。
- [ ] 现有 `content_create` 图片生成相关测试通过。
- [ ] 生成作品原创配图链路不再把空 prompt 打到远端接口。

---

## 2. 方案

### 技术方案
在内容创作提示词归一化层校验 `cover_prompt` 非空；在图片集成层 `generate_images()` 入口逐项校验 prompt 列表，任意提示词为空时直接抛出本地 `StoreError`。保持 `uni` provider 的小写 `prompt` payload 契约不变，同步补充回归测试，确保空 prompt 不会触发远程请求或导致图片槽位错位。

### 影响范围
```yaml
涉及模块:
  - integrations: 图片生成 provider 请求体构建
  - tests: 图片生成回归测试
预计变更文件: 2
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 远端错误文案容易被误判为字段大小写问题 | 中 | 保持 Uni 官方示例的小写 `prompt`，只修本地空值校验 |
| 过滤空配图提示词后生成图片数量可能少于原始列表 | 低 | `cover_prompt` 必须非空；空内页提示词本身不可生成，跳过比远端 400 更可控 |

## 4. 核心场景

### 场景: 原创配图使用 Uni 文生图
**模块**: integrations
**条件**: 租户或系统配置 `IMAGE_PROVIDER=uni`，生成作品工作流进入原创配图节点。
**行为**: `generate_images()` 为每条配图提示词构建 Uni 文生图请求并调用 `/images/generations`。
**结果**: 非空提示词使用小写 `prompt` 字段发送；空提示词在本地被拦截，避免远端 `field Prompt is required` 参数错误。

## 5. 技术决策

### fix-uni-image-prompt-field#D001: 保持 Uni 小写 prompt，并在本地拦截空值
**日期**: 2026-04-28
**状态**: ✅采纳
**背景**: Uni 官方兼容 OpenAI 的 `/v1/images/generations` 示例使用小写 `prompt`。远端错误 `field Prompt is required` 更可能来自空提示词或下游服务结构字段名，而不是请求字段大小写。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 保持小写 `prompt`，补充本地空值校验 | 符合官方示例，能提前暴露真实问题，影响面小 | 不会掩盖上游提示词生成失败 |
| B: 将 Uni 字段改为大写 `Prompt` | 表面贴合错误文案 | 与官方示例冲突，可能引入新的 400 |
**决策**: 选择方案 A
**理由**: 用户提供的 curl 示例证明小写 `prompt` 是正确协议；应修复本地空值进入远端的问题。
**影响**: 影响内容创作配图提示词校验与所有 provider 的空 prompt 防护，不改变 provider payload 协议。
