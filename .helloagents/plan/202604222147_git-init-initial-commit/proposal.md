## 1. 需求

### 背景
当前项目目录尚未初始化 Git 仓库，无法追踪代码历史或进行后续协作。用户要求在当前目录初始化仓库并完成首次提交，同时仅排除 `.venv/` 与 `var/`，其余现有文件全部纳入版本控制。

### 目标
- 初始化当前目录的 Git 仓库
- 通过最小 `.gitignore` 排除 `.venv/` 与 `var/`
- 将其余现有文件纳入首个提交
- 完成一次可用的初始 commit

### 约束条件
```yaml
时间约束: 无
性能约束: 无
兼容性约束: 不调整现有项目代码行为
业务约束: 按用户要求保留 .env、.helloagents 等文件进入版本控制，仅排除 .venv/ 与 var/
```

### 验收标准
- [ ] 当前目录成功初始化 `.git`
- [ ] `.venv/` 与 `var/` 不进入暂存区
- [ ] 首次提交成功创建
- [ ] 提交后工作区处于干净状态

---

## 2. 方案

### 技术方案
- 新增最小 `.gitignore`，只忽略 `.venv/` 与 `var/`
- 初始化 Git 仓库
- 将当前其余文件全部纳入暂存区
- 使用 `Initial commit` 作为首个提交信息完成提交

### 影响范围
```yaml
涉及模块:
  - 仓库根目录: 新增 .gitignore，初始化 .git 元数据
  - .helloagents: 新增本次初始化方案包记录
预计变更文件: 4
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| `.env` 含敏感信息并进入历史 | 高 | 按用户明确要求执行，并在方案中记录 |
| `.helloagents` 内容进入首个提交 | 中 | 按用户范围执行，不做额外排除 |

### 场景: 首次建立版本控制
**模块**: 仓库根目录
**条件**: 当前目录不存在 `.git`
**行为**: 写入最小 `.gitignore`，初始化 Git 仓库并提交现有文件
**结果**: 当前项目具备可追踪的首次版本历史

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### git-init-initial-commit#D001: 首次提交严格遵循用户指定排除范围
**日期**: 2026-04-22
**状态**: ✅采纳
**背景**: 用户明确要求仅排除 `.venv/` 与 `var/`，其余内容全部提交，即使其中包含 `.env` 与 `.helloagents/`。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 按常规安全策略额外排除 `.env`、`__pycache__/`、`.helloagents/` | 更安全，更常见 | 不符合用户明确范围 |
| B: 严格只排除 `.venv/` 与 `var/` | 完全符合用户要求 | 会把 `.env` 等本地内容写入历史 |
**决策**: 选择方案 B
**理由**: 本任务目标是按用户明确边界初始化仓库，不额外替用户改变提交范围。
**影响**: 首次提交将包含 `.env` 与 `.helloagents/` 等本地文件。
